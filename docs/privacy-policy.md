# Privacy Policy for WebLang Translator

**Last Updated:** March 14, 2026

## Overview

WebLang Translator is a browser extension that translates page content, supports quick translation of selected text, and optionally provides AI analysis and text-to-speech features. This policy explains what data the extension accesses, stores, and sends to third-party services.

The developer of WebLang Translator does not operate a backend server for this extension. Extension processing happens in your browser and through the third-party services you choose to use.

## What Data the Extension Accesses

WebLang Translator may access the following data when you use its features:

- Text from web pages you open in your browser, for page translation and quick translation
- Text you select manually for quick translation
- Translated text generated by the extension
- Text you choose to send to AI features such as summarize, analyze, keywords, or Gemini text-to-speech
- Settings you configure in the extension, including provider selection, language preferences, UI preferences, and feature toggles
- API keys that you enter for supported services such as Google, Azure, or Gemini

The extension is not designed to collect personally identifiable information for the developer. However, the text you choose to translate or analyze may contain personal or sensitive information depending on the website you are using.

## What We Store

### Browser Storage

The extension stores settings in Chrome extension storage, specifically `chrome.storage.sync`. Depending on your browser settings, this data may be synchronized by the browser vendor across devices signed into the same browser account.

Stored data may include:

- Source and target language preferences
- Translation mode and quick-translate settings
- Display preferences such as theme and translation color
- Tooltip and AI/TTS feature toggles
- Selected translation provider
- Custom Gemini model and Gemini voice preferences
- API keys you enter for Google, Azure, or Gemini
- The most recent AI result and related metadata used by the extension's export flow

The developer does not receive this stored data on a separate server.

### Temporary Session Storage

For performance, page translation results may be cached temporarily in the active tab using `sessionStorage`. This cache is used only inside the current browser tab/session and is not intended as long-term storage.

### Local Browser Actions

If you use export or copy features:

- Exported files are created locally in your browser and saved only when you choose to export
- Copy actions write text to your clipboard only when you click the copy control

## What We Send to Third Parties

### Translation Requests

When you use translation features, the text being translated may be sent to one or more translation providers, depending on your settings and fallback behavior. Supported providers in the current project include:

- Google Cloud Translation API
- Google Translate web endpoints
- Microsoft Azure Translator
- MyMemory
- LibreTranslate

In addition to the text itself, requests may include technical parameters needed to perform the translation, such as source language, target language, and your API key when a paid provider is configured.

### AI and Text-to-Speech Requests

If you enable and use AI features, translated text or selected text may be sent to Google Gemini for:

- Summaries
- Analysis
- Keyword extraction
- Gemini-powered text-to-speech generation

These requests may include technical parameters such as language, model name, selected voice, and your Gemini API key.

### No Developer Backend

WebLang Translator does not send your data to a server controlled by the developer. Data is sent only to the third-party services needed to fulfill the feature you actively use.

## Data Sharing and Sale

We do not sell user data.

We do not transfer or share user data with the developer's own servers because this project does not use a developer-operated backend.

When you use translation, AI, or Gemini TTS features, the relevant text and required request metadata are transmitted to the selected third-party provider so the feature can work.

## Permissions Used

| Permission or Access | Purpose |
|---|---|
| `storage` | Save extension settings and recent AI output in browser storage |
| `activeTab` | Allow user-initiated access to the current tab for fallback script injection |
| `scripting` | Inject scripts or CSS needed for translation and AI UI behavior |
| `downloads` | Let the user export translation or AI results as files |
| `sidePanel` | Display the extension interface in Chrome's side panel |
| Website access on `http` and `https` pages | Read page text and modify the page to show translations and quick-translate UI |
| Host permissions for provider domains | Send requests to translation, AI, and TTS service endpoints used by the extension |

## Third-Party Services

If you use the relevant features, your data may be processed by the third-party provider you select or trigger through fallback behavior. Those providers may handle data according to their own terms and privacy policies.

Examples of providers used by this project include Google services, Microsoft Azure Translator, MyMemory, and LibreTranslate.

## Data Retention

- Settings remain in browser extension storage until you change them, clear browser extension data, or uninstall the extension
- Session translation cache is temporary and tied to the current tab/session
- Exported files remain wherever you choose to save them

## Security

Data stored by the extension relies on the browser's extension storage and sandboxing model. No system can guarantee absolute security, and third-party providers process requests under their own infrastructure and policies.

## Changes to This Policy

We may update this privacy policy from time to time. Any changes will be reflected in the "Last Updated" date above.
